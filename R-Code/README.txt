#Bachelor Thesis: Frequentist and Bayesian Logistic Regression for Heart Disease Prediction

This folder contains all R Markdown scripts used to reproduce the analyses presented in the thesis. The scripts implement a complete workflow consisting of data preprocessing, multiple imputation, frequentist and Bayesian logistic regression modelling, model comparison, cross-validation, and subsampling analyses.

The workflow was originally developed for a heart disease dataset but can also be adapted to other binary classification datasets with appropriate modifications.

Required dataset:

heart disease dataset.csv

Main scripts:

* Exploratory Analysis.Rmd
* Imputation code.Rmd
* Frequentist Logistic Regression & pooling.Rmd
* Bayesian Logistic Regression & pooling.Rmd
* Model Comparison.Rmd
* Cross Validation.Rmd
* Subsampling.Rmd

Generated folders:

* Imputed-Data
* Results
* figures
* Tables

These folders are automatically created when the scripts are executed.

======================================================================
COMPLETE EXECUTION ORDER
========================

The scripts should be run in the following order.

STEP 1

Exploratory Analysis.Rmd

Purpose:
Perform exploratory data analysis and obtain descriptive summaries of the dataset.

Main outputs:

* Dataset summary tables
* Missing-value summaries
* Numerical variable distributions
* Categorical variable distributions
* Predictor-versus-outcome visualisations

Required input:

* heart disease dataset.csv

Can be skipped:
Yes. This script is not required for the modelling workflow but is recommended for understanding the dataset.

STEP 2

Imputation code.Rmd

Purpose:
Prepare the data and perform Multiple Imputation by Chained Equations (MICE).

Main outputs:

* imputed_data_m5.rds
* imputed_data_m10.rds
* imputed_data_m20.rds
* completed imputed datasets
* imputation diagnostic plots

Required input:

* heart disease dataset.csv

Required before:

* Frequentist Logistic Regression & pooling.Rmd
* Bayesian Logistic Regression & pooling.Rmd
* Cross Validation.Rmd
* Subsampling.Rmd

STEP 3

Frequentist Logistic Regression & pooling.Rmd

Purpose:
Fit frequentist logistic regression models across multiple imputations and pool the results using Rubin's Rules.

Main outputs:

* Full model results
* Backward-selection results
* Final model results
* Odds ratios and confidence intervals
* Predicted probabilities
* Predictive performance summaries

Required input:

* imputed_data_m5.rds
* imputed_data_m10.rds
* imputed_data_m20.rds

STEP 4

Bayesian Logistic Regression & pooling.Rmd

Purpose:
Perform Bayesian variable selection and fit Bayesian logistic regression models using Hamiltonian Monte Carlo (HMC).

Main outputs:

* Posterior inclusion probabilities (PIPs)
* Final selected Bayesian model
* Posterior odds ratios
* Credible intervals
* Predicted probabilities
* HMC convergence diagnostics
* Predictive performance summaries

Required input:

* imputed_data_m5.rds
* imputed_data_m10.rds

STEP 5

Model Comparison.Rmd

Purpose:
Compare the final frequentist and Bayesian models.

Main outputs:

* Odds-ratio comparison plots
* Predicted-probability comparison plots
* Variable-selection comparison table

Required input:

* Outputs generated by Steps 3 and 4

STEP 6

Cross Validation.Rmd

Purpose:
Evaluate predictive performance using cross-validation.

Main outputs:

* Stratified 10-fold cross-validation results
* ROC curves
* AUC summaries
* Accuracy summaries
* Sensitivity summaries
* Specificity summaries
* Runtime summaries

Required input:

* imputed_data_m5.rds

STEP 7

Subsampling.Rmd

Purpose:
Evaluate model stability under reduced sample sizes.

Main outputs:

* Variable inclusion frequencies
* Predictive performance summaries
* Odds-ratio stability summaries
* Runtime stability summaries
* Inclusion-frequency figures

Required input:

* imputed_data_m5.rds

======================================================================
USING THE WORKFLOW WITH A DIFFERENT DATASET
===========================================

The workflow can be adapted to other binary classification datasets.

To use a different dataset:

1. Replace the original dataset with your own dataset.

2. Update the dataset path in:

   * Exploratory Analysis.Rmd
   * Imputation code.Rmd

3. Replace the response variable throughout the scripts.

4. Update all predictor names used in:

   * Frequentist Logistic Regression & pooling.Rmd
   * Bayesian Logistic Regression & pooling.Rmd
   * Cross Validation.Rmd
   * Subsampling.Rmd

5. Modify the factor-conversion section in:

   * Exploratory Analysis.Rmd
   * Imputation code.Rmd

6. Review the imputation settings and modify them if the missing-data structure differs from the original dataset.

7. Update the model formulas used in the frequentist and Bayesian analyses.

The workflow assumes:

* a binary response variable,
* logistic regression as the modelling framework,
* multiple imputation for missing-data handling.

======================================================================
MAIN FUNCTIONS AND EXECUTION ORDER
==================================

Users do not normally need to call functions manually. Running the scripts in the recommended order automatically executes the required functions.

The following section describes the main functions used in each script, the order in which they are executed, and their role within the overall workflow.

Imputation code.Rmd

Main function:

run_mice()

Purpose:

Generates multiple completed datasets using Multiple Imputation by Chained Equations (MICE).

Outputs:

* imputed_data_m5.rds
* imputed_data_m10.rds
* imputed_data_m20.rds
* completed datasets
* imputation diagnostics

Execution order:

This is the first modelling function executed in the workflow and must be completed before any modelling scripts are run.

Frequentist Logistic Regression & pooling.Rmd

Functions executed:

1. backward_selection()

Purpose:

Identifies important predictors using backward elimination based on pooled p-values.

2. pool_selected_models()

Purpose:

Fits the final logistic regression model across all imputed datasets and pools parameter estimates using Rubin's Rules.

Outputs:

* selected predictors
* pooled coefficients
* odds ratios
* confidence intervals
* predicted probabilities
* predictive performance summaries

Execution order:

This script should be run immediately after multiple imputation.

Bayesian Logistic Regression & pooling.Rmd

Functions executed:

1. run_spike_slab_selection()

Purpose:

Computes posterior inclusion probabilities and identifies important predictors.

2. fit_bayesian_model()

Purpose:

Fits Bayesian logistic regression models using Hamiltonian Monte Carlo.

3. fit_final_bayesian_model()

Purpose:

Fits the final Bayesian model using the selected predictors.

Outputs:

* posterior inclusion probabilities
* selected predictors
* posterior odds ratios
* credible intervals
* predicted probabilities
* HMC diagnostics

Execution order:

This script should be run after the frequentist analysis has been completed.

Model Comparison.Rmd

Purpose:

Uses the outputs from the frequentist and Bayesian analyses to compare the final models.

Required before running:

* Frequentist Logistic Regression & pooling.Rmd
* Bayesian Logistic Regression & pooling.Rmd

Outputs:

* odds-ratio comparison plots
* predicted-probability comparison plots
* variable-selection comparison table

Execution order:

Run after both modelling scripts have been completed.

Cross Validation.Rmd

Functions executed:

1. run_frequentist_cv()

Purpose:

Evaluates predictive performance of the frequentist model using stratified cross-validation.

2. run_bayesian_cv()

Purpose:

Evaluates predictive performance of the Bayesian model using stratified cross-validation.

Outputs:

* AUC
* Accuracy
* Sensitivity
* Specificity
* ROC curves
* runtime summaries

Execution order:

Run after the final frequentist and Bayesian models have been obtained.

Subsampling.Rmd

Functions executed:

1. run_frequentist_subsampling()

Purpose:

Evaluates frequentist variable-selection stability under reduced sample sizes.

2. run_bayesian_subsampling_selection()

Purpose:

Evaluates Bayesian variable-selection stability under reduced sample sizes.

3. run_frequentist_predictive_performance()

Purpose:

Evaluates predictive performance under repeated subsampling.

4. run_bayesian_predictive_performance()

Purpose:

Evaluates Bayesian predictive performance under repeated subsampling.

Outputs:

* inclusion frequencies
* predictive performance summaries
* odds-ratio stability summaries
* runtime summaries

Execution order:

Run after the final models have been fitted and validated.

======================================================================
NOTES
=====

* All file paths are relative and should work on any operating system provided the repository structure is preserved.
* Output folders are created automatically when required.
* Bayesian analyses may require substantial computation time due to Hamiltonian Monte Carlo sampling and spike-and-slab variable selection.
* Running the scripts in the recommended order ensures that all required intermediate files are available for subsequent analyses.
* The repository is intended both as a replication package for the thesis and as a reusable template for binary logistic regression analyses involving missing data, multiple imputation, frequentist inference, Bayesian inference, cross-validation, and subsampling.
